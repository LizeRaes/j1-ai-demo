Scope
This policy governs refunds for appointment-related charges processed through MedicalAppointment. It applies to all billing-related support requests.

Definitions
Appointment Time: The scheduled start time of the appointment.
Cancellation Window: The period ending 24 hours before the Appointment Time.
System Error: A confirmed application behavior that prevents a supported user action.
Duplicate Charge: More than one charge associated with the same appointment without user intent.

Refund Eligibility

Refunds may be approved when ALL of the following conditions are met:

The user attempted a supported action within the cancellation window
AND

The action failed due to a system error
AND

The appointment did not take place

Examples of eligible scenarios include:

The cancel or reschedule button was unavailable before the cancellation window closed

The application created two appointments or charges for the same booking

A confirmed application defect caused incorrect billing status

Non-Refundable Scenarios

Refunds must NOT be issued in the following cases:

The user missed the appointment

The cancellation request was made less than 24 hours before the Appointment Time

The user changed their mind after booking

The appointment was successfully completed

No system error can be confirmed

Agent Handling Rules

Support agents may:

Acknowledge refund requests

Explain the refund policy

Collect relevant details (appointment ID, date, issue description)

Support agents must NOT:

Promise a refund

Approve or deny refunds directly

Suggest that refunds are automatic

Bypass internal review

Decision Authority

Refund approval requires internal review by the Billing Team.
All approved refunds must be logged with a reason code matching this policy.

User Communication

All billing and refund communications must use approved response templates.
Agents must avoid speculative language and must not disclose internal billing mechanisms.

End of Policy
